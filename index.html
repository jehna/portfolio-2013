Black Powder

Black Powder is a game available for iOS devices as a free ad-funded version and as a single-purchase premium version. The game takes place at the coast of Hamina, letting the player defend his homeland as Powder-Pete.

Highlight: Enemy AI

Ship behaviour is a big part of the game; Ships have to be predictable, but smart enough to give a decent challenge to the player.

I implemented a grid-like ship route system, that was customizable between different maps. Routes helped ship AI to take shortcuts or detours on their route.

Roasty rooster

Roasty Rooster is an upcoming action game for iOS, Android and OUYA.

The game had it's first prototype at early 2012. One year later it got a viral hit, when a video of OUYA running an early dev version of Roasty Rooster got 200k+ views on youtube.

Highlight: Font system

Unity doesn't support bitmap fonts, so I needed to implement a system to display pixel-perfect bitmap fonts.

Because I love doing plugins and editors, it was natural to do something that other developers could also use. I programmed SpriteText, a plugin that's available for everyone on Asset Store.

Saari 42

Saari 42 is an educative game for forthcoming employees. By telling a story it teaches the basics of getting a job of your dreams.

The game was done in cooperation with Kouvola Region Vocational College, as a final year project for some of the students.

Highlight: Pathfinding system

At the time I was making the game, there was no ready-to-use pathfinding systems in Unity.

To originally prevent player from getting stuck on some difficult places, I programmed a simple 3D-world pathfinding system that used waypoints to determine the best possible route.

Highlight: Storyboard system

The game included a lot of dialogue between characters, so an easy-to-use storyboard engine had to be programmed. The linear story had to be easily debugged and tested at any given point of the story.

This was a really fun project to see how a linear story RPG game can be built. I'm definitely looking forward to the next opportunity to build a RPG game.

The Whimper

Game made at Finnish Game Jam 2013. It was built in 48 hours with a total of five people.

We wanted to do a co-op dungeon escape horror game that would be easily ported to forthcoming OUYA console. The gameplay wasn't as finished as it could be, but the game succeeded to freak the crap out of me couple of times. Mission successful.

Highlight: Procedural maps

My responsibility in the game was to build a random level generator. I did my background search and the result was one of the programs that I'm the most proud of.

The level generator consists from different procedural "builders". E.g. the first builder usually does one big main route for the map, which is followed by multiple sub-route builders. Subsequent builders will generate rooms of different sizes, place the players on the map and finally decorate the map.

Putt-e

Failing is a big part of design process. Fast failing. So how fast could I make a game?

Answer: In 12 hours. Putt-e was a personal challenge to make a full game as fast as possible.

The first objective was to do a fully playable game in 6 hours, but i found out the hard way that boolean modifiers are damn hard to implement on 3D objects. So overall 12 hours included everything from designing to trying to total failing to redesigning and actually succeeding.
